# QA Portfolio

Professional quality assurance documentation samples for the NovaCart e-commerce demo platform. Content is structured for portfolio review and reflects industry-standard QA artifacts.

## Repository Structure

| Folder | Description |
|--------|-------------|
| [Test-Plans](Test-Plans/) | Formal test plans defining scope, strategy, and schedule |
| [Test-Cases](Test-Cases/) | Detailed test cases with steps and expected results |
| [Bug-Reports](Bug-Reports/) | Defect reports with reproduction steps and impact analysis |
| [RTM](RTM/) | Requirements Traceability Matrix linking reqs to tests |
| [API-Test-Scenarios](API-Test-Scenarios/) | API endpoint test scenarios and expected responses |
| [Release-Checklist](Release-Checklist/) | Pre and post-release verification checklists |

## Sample Project

All sample documents reference **NovaCart**, a fictional B2C e-commerce application modeled after publicly available QA training platforms. Scenarios cover authentication, checkout, API validation, and release management workflows.
